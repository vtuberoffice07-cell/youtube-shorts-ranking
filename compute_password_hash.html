🔐 パスワードハッシュ生成

このページの用途: viewer.html のパスワード認証を再有効化する際の SHA-256 ハッシュを計算するためのローカルユーティリティ。
入力したパスワードはどこにも送信されず、ブラウザ内（Web Crypto API）で計算される。

viewer.html の貼り付け箇所（参考）

const PASS_SALT = 'vtbuzz-2026-04';
const PASS_HASH_HEX = '...ここに上記のハッシュを貼り付け...';

3. 再有効化の手順

  1. このページで新しいパスワードのハッシュを計算する
  2. viewer.html を開いて 2 箇所を編集する:
    • const AUTH_REQUIRED = false; → true に戻す
    • const PASS_HASH_HEX = '...'; → 上記のハッシュに置き換え
  3. 必要なら PASS_SALT も別の値に変える（このページの「ソルト」入力と同期させる）
  4. commit & push して GitHub Pages に反映
  5. シークレット/キャッシュをクリアしてログイン確認
制約: フロントエンド認証は本質的に obfuscation。JavaScript を読まれればハッシュもソルトも見えるため、 辞書攻撃やレインボーテーブルで弱いパスワード（例: vtuber、123456）は逆算可能。
本格的に限定公開したい場合: GitHub リポジトリ Private 化 + GitHub Pages Private 化が必要。
